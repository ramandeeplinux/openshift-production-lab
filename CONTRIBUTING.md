# Contributing

Thank you for your interest in contributing to this repository.

This project demonstrates a production-style Red Hat OpenShift Platform Engineering Lab. Contributions that improve documentation, examples, manifests, or automation are welcome.

## Contribution Workflow

1. Fork the repository.
2. Create a new feature branch.
3. Make your changes.
4. Test your changes.
5. Commit using a meaningful commit message.
6. Push your branch.
7. Open a Pull Request.

## Guidelines

- Follow the existing folder structure.
- Keep documentation clear and consistent.
- Validate Kubernetes and OpenShift manifests before committing.
- Update README files when adding new content.
- Use descriptive commit messages.

Thank you for helping improve this repository.
